# src

Aquí se almacena el **código fuente** principal y todos los módulos, scripts y utilidades desarrollados para la plataforma ICARIA.

## Contenido sugerido

- Scripts de procesamiento de datos.
- Módulos de validación de condiciones/casos.
- Subcarpetas por módulos funcionales, si lo requiere la escala del proyecto.
- Utilidades para conexión con fuentes externas.
- Código reutilizable, helper functions.

## Buenas prácticas

- Mantener código documentado y modular.
- Implementar pruebas unitarias, alojando los tests en una subcarpeta `/tests` o similar.
- Separar lógica de negocio y utilidades comunes.
